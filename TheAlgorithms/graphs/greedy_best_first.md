# 탐욕적 최선 우선 탐색 (Greedy Best-First Search)

이 문서는 `greedy_best_first.py` 파일에 구현된 **탐욕적 최선 우선 탐색(Greedy Best-First Search)** 알고리즘에 대해 설명합니다.

## 개요

탐욕적 최선 우선 탐색은 그래프 탐색 알고리즘의 일종으로, 각 단계에서 목표 지점에 가장 가깝다고 판단되는(가장 유망한) 노드를 선택하여 탐색을 진행합니다. A* 알고리즘과 유사하지만, 시작점부터 현재 노드까지의 실제 비용($g(n)$)은 무시하고, 현재 노드에서 목표까지의 추정 비용($h(n)$)만을 평가 함수($f(n) = h(n)$)로 사용합니다.

이 코드는 2차원 그리드 상에서 장애물을 피해 시작점에서 목표점까지 이동하는 경로를 찾습니다.

## 주요 설정

- `grid`: 0(이동 가능)과 1(장애물)로 구성된 2차원 리스트입니다.
- `delta`: 상, 좌, 하, 우 4방향 이동을 정의합니다. `[[-1, 0], [0, -1], [1, 0], [0, 1]]`

## 주요 클래스

### `Node`

- **목적**: 그리드 상의 특정 지점(노드)에 대한 정보를 저장합니다.
- **속성**:
  - `pos_x`, `pos_y`: 현재 좌표.
  - `goal_x`, `goal_y`: 목표 좌표.
  - `g_cost`: 시작점부터 현재까지의 비용 (이 알고리즘에서는 우선순위 결정에 직접적으로 사용되지 않음).
  - `f_cost`: 목표까지의 추정 비용 (휴리스틱 값).
  - `parent`: 경로 역추적을 위한 부모 노드.
- **메서드**:
  - `calculate_heuristic()`: 맨해튼 거리(Manhattan Distance, $|x_1 - x_2| + |y_1 - y_2|$)를 계산하여 반환합니다.
  - `__lt__`: 노드 간의 비교 연산자입니다. `f_cost`가 작은 노드가 더 높은 우선순위를 가집니다.

### `GreedyBestFirst`

- **목적**: 탐욕적 최선 우선 탐색 알고리즘을 실행합니다.
- **속성**:
  - `open_nodes`: 탐색할 노드들을 저장하는 리스트 (우선순위 큐 역할).
  - `closed_nodes`: 이미 탐색이 완료된 노드들을 저장하는 리스트.
- **주요 메서드**:
  - `search()`:
    - `open_nodes`가 빌 때까지 반복합니다.
    - `f_cost`가 가장 낮은 노드를 꺼내 현재 노드로 설정합니다.
    - 목표에 도달하면 `retrace_path`를 호출하여 경로를 반환합니다.
    - 현재 노드의 이웃 노드(`successors`)를 찾아 `open_nodes`에 추가합니다.
  - `get_successors(parent)`:
    - 현재 위치에서 상하좌우로 이동 가능한 이웃 노드들을 생성하여 반환합니다.
    - 그리드 범위를 벗어나거나 장애물(1)인 경우는 제외합니다.
  - `retrace_path(node)`:
    - 목표 노드에서부터 부모 노드를 따라 거슬러 올라가며 전체 경로를 복원합니다.

## 사용법

`if __name__ == "__main__":` 블록에서 실행 예시를 확인할 수 있습니다.

1. 시작점 `(0, 0)`과 목표점(그리드의 우하단 끝)을 설정합니다.
2. `GreedyBestFirst` 객체를 생성하고 `search()` 메서드를 호출합니다.
3. 경로가 발견되면, 해당 경로상의 위치를 그리드에 `2`로 표시하여 출력합니다.

```python
# 실행 결과 예시 (경로가 2로 표시됨)
[2, 0, 0, 0, 0, 0, 0]
[2, 1, 0, 0, 0, 0, 0]
...
```
