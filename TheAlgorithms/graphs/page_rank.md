# 페이지랭크 알고리즘 (PageRank Algorithm)

이 문서는 `page_rank.py` 파일에 구현된 **페이지랭크(PageRank)** 알고리즘에 대해 설명합니다.

## 개요

페이지랭크는 구글 검색 엔진의 초기 버전에 사용된 알고리즘으로, 웹 페이지의 중요도를 측정하기 위해 링크 구조를 분석합니다. 어떤 페이지가 다른 중요한 페이지들로부터 링크를 많이 받을수록 그 페이지의 중요도(랭크)가 올라간다는 아이디어에 기반합니다.

## 주요 클래스: `Node`

### `__init__(self, name)`
- **목적**: 그래프의 노드(웹 페이지)를 초기화합니다.
- **속성**:
  - `name`: 노드의 이름.
  - `inbound`: 이 노드로 들어오는 링크를 가진 노드들의 이름 리스트.
  - `outbound`: 이 노드에서 나가는 링크가 연결된 노드들의 이름 리스트.

### 메서드
- `add_inbound(node)`: 들어오는 링크를 추가합니다.
- `add_outbound(node)`: 나가는 링크를 추가합니다.
- `__repr__(self)`: 노드의 상태(이름, 인바운드, 아웃바운드)를 문자열로 반환합니다.

## 주요 함수: `page_rank`

### `page_rank(nodes, limit=3, d=0.85)`

- **목적**: 주어진 노드들에 대해 페이지랭크 알고리즘을 실행하여 각 노드의 랭크를 계산합니다.
- **매개변수**:
  - `nodes`: `Node` 객체들의 리스트.
  - `limit`: 알고리즘 반복 횟수 (기본값: 3).
  - `d`: 댐핑 팩터(Damping Factor). 사용자가 링크를 클릭하다가 지루해져서 다른 페이지로 이동할 확률을 반영합니다 (기본값: 0.85).
- **알고리즘 동작**:
  1. **초기화**: 모든 노드의 랭크를 1로 설정합니다.
  2. **반복**: 지정된 `limit` 횟수만큼 반복합니다.
     - 각 노드에 대해 다음 공식을 사용하여 랭크를 갱신합니다.
     - $PR(A) = (1 - d) + d \sum_{i=1}^{n} \frac{PR(T_i)}{C(T_i)}$
       - $PR(A)$: 페이지 A의 랭크.
       - $PR(T_i)$: 페이지 A로 링크를 건 페이지 $T_i$의 랭크.
       - $C(T_i)$: 페이지 $T_i$가 가진 전체 아웃바운드 링크 수.
  3. **출력**: 매 반복마다 갱신된 랭크 점수를 딕셔너리 형태로 출력합니다.

## 실행 흐름 (`main` 함수)

1. 사용자로부터 노드 이름들을 입력받습니다 (예: `A B C`).
2. 입력받은 이름으로 `Node` 객체들을 생성합니다.
3. 코드 상단에 정의된 인접 행렬(`graph`)을 사용하여 노드 간의 연결 관계(링크)를 설정합니다.
   - `graph` 변수는 A, B, C 세 노드 간의 연결을 정의하고 있습니다.
4. `page_rank` 함수를 호출하여 계산을 수행합니다.

## 사용법

`if __name__ == "__main__":` 블록을 통해 스크립트를 실행할 수 있습니다.

**주의 사항**:
- 이 코드는 `graph` 변수에 하드코딩된 인접 행렬(3x3)을 사용하므로, 입력받는 노드의 이름 개수(3개)와 순서가 이 행렬과 일치해야 의도한 대로 동작합니다.
