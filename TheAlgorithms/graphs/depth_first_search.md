# 깊이 우선 탐색 (Depth First Search)

이 문서는 `depth_first_search.py` 파일에 구현된 **비재귀적 깊이 우선 탐색(Non-recursive DFS)** 알고리즘에 대해 설명합니다.

## 개요

깊이 우선 탐색(DFS)은 그래프의 시작 정점에서 출발하여 한 분기를 끝까지 탐색한 후, 다시 돌아와 다른 분기를 탐색하는 방식입니다. 이 구현에서는 재귀 호출 대신 **스택(Stack)** 자료구조를 사용하여 구현되었습니다.

## 주요 함수: `depth_first_search`

### `depth_first_search(graph: dict, start: str) -> set[str]`

- **목적**: 주어진 시작 정점(`start`)에서부터 그래프(`graph`)를 깊이 우선으로 탐색하여 방문한 모든 정점들의 집합을 반환합니다.
- **매개변수**:
  - `graph`: 인접 리스트 형태의 딕셔너리. 키는 정점, 값은 이웃 정점들의 리스트입니다.
  - `start`: 탐색을 시작할 정점 (문자열).
- **반환값**: 방문한 정점들이 포함된 집합(`set`).

### 알고리즘 동작 원리

1. **초기화**:
   - `explored`: 방문한 정점을 저장하는 집합. 시작 정점을 포함하여 초기화합니다.
   - `stack`: 탐색할 정점을 저장하는 스택(리스트). 시작 정점을 포함하여 초기화합니다.

2. **탐색 루프 (While Stack)**:
   - 스택이 빌 때까지 다음 과정을 반복합니다.
   - **Pop**: 스택의 **마지막 요소**(가장 최근에 추가된 요소) `v`를 꺼냅니다.
   - **Visit**: `v`를 `explored` 집합에 추가합니다.
   - **Push Neighbors**: `v`의 이웃 정점들을 확인합니다.
     - `reversed(graph[v])`를 사용하여 역순으로 순회합니다. 이는 스택의 LIFO(Last-In First-Out) 특성상, 나중에 넣은 것이 먼저 나오므로 원래 순서대로 방문하기 위함입니다.
     - 아직 방문하지 않은(`explored`에 없는) 이웃 정점이라면 스택에 추가(`append`)합니다.

### BFS와의 차이점
- **자료구조**: BFS는 큐(Queue)를 사용하여 먼저 들어온 요소를 먼저 처리(FIFO)하지만, 이 DFS 구현은 스택(Stack)을 사용하여 나중에 들어온 요소를 먼저 처리(LIFO)합니다.
- **탐색 순서**: BFS는 가까운 노드부터 넓게 탐색하는 반면, DFS는 한 방향으로 깊게 탐색합니다.

## 사용법

`if __name__ == "__main__":` 블록에서 `doctest`를 실행하여 코드를 검증하고 예제를 실행합니다.

```python
G = {
    "A": ["B", "C", "D"],
    "B": ["A", "D", "E"],
    # ... (생략)
}

print(depth_first_search(G, "A"))
# 출력: {'A', 'B', 'C', 'D', 'E', 'F', 'G'} (집합이므로 순서는 보장되지 않음)
```
