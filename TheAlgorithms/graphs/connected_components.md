# 연결 요소 (Connected Components)

이 문서는 `connected_components.py` 파일에 구현된 **연결 요소(Connected Components)** 찾기 알고리즘에 대해 설명합니다.

## 개요

그래프 이론에서 연결 요소란 무향 그래프의 부분 그래프로서, 해당 부분 그래프 내의 모든 정점 쌍 사이에 경로가 존재하고, 이 부분 그래프에 속하지 않는 정점과는 연결되지 않은 최대의 부분 그래프를 의미합니다.

이 코드는 **깊이 우선 탐색(DFS)**을 사용하여 그래프 내의 모든 연결 요소를 찾아냅니다.

## 주요 함수

### `dfs(graph: dict, vert: int, visited: list) -> list`

- **목적**: 깊이 우선 탐색을 수행하여 시작 정점(`vert`)과 연결된 모든 정점을 찾습니다.
- **매개변수**:
  - `graph`: 인접 리스트 형태의 딕셔너리.
  - `vert`: 탐색을 시작할 정점 인덱스.
  - `visited`: 각 정점의 방문 여부를 저장하는 리스트.
- **반환값**: 시작 정점과 연결된 정점들의 리스트 (하나의 연결 요소).
- **동작**:
  - 현재 정점을 방문 처리합니다.
  - 인접한 정점 중 방문하지 않은 정점에 대해 재귀적으로 `dfs`를 호출합니다.
  - 방문한 모든 정점을 리스트로 합쳐서 반환합니다.

### `connected_components(graph: dict) -> list`

- **목적**: 그래프 전체를 순회하며 모든 연결 요소를 찾습니다.
- **매개변수**:
  - `graph`: 인접 리스트 형태의 딕셔너리.
- **반환값**: 연결 요소들의 리스트 (리스트의 리스트).
- **동작**:
  - `visited` 리스트를 초기화합니다.
  - 그래프의 모든 정점에 대해 반복합니다:
    - 만약 정점이 아직 방문되지 않았다면, 새로운 연결 요소의 시작점이므로 `dfs`를 호출합니다.
    - `dfs` 결과(하나의 연결 요소)를 `components_list`에 추가합니다.

## 사용법

`if __name__ == "__main__":` 블록에서 `doctest`를 실행하여 코드를 검증합니다.

코드 내에 정의된 테스트 그래프 예시:
```python
test_graph_1 = {0: [1, 2], 1: [0, 3], 2: [0], 3: [1], 4: [5, 6], 5: [4, 6], 6: [4, 5]}
# 결과: [[0, 1, 3, 2], [4, 5, 6]]

test_graph_2 = {0: [1, 2, 3], 1: [0, 3], 2: [0], 3: [0, 1], 4: [], 5: []}
# 결과: [[0, 1, 3, 2], [4], [5]]
```
