# 인접 행렬 그래프 (Graph Adjacency Matrix)

이 문서는 `graph_matrix.py` 파일에 구현된 **인접 행렬(Adjacency Matrix)** 방식의 그래프 자료구조에 대해 설명합니다.

## 개요

인접 행렬은 그래프를 표현하는 가장 기본적인 방법 중 하나로, $V \times V$ 크기의 2차원 배열(행렬)을 사용하여 정점 간의 연결 관계를 저장합니다. 이 구현은 무향 그래프(Undirected Graph)를 다룹니다.

## 주요 클래스: `Graph`

### `__init__(self, vertex)`

- **목적**: 그래프 객체를 초기화합니다.
- **매개변수**:
  - `vertex`: 정점의 개수 ($V$).
- **동작**:
  - `vertex` $\times$ `vertex` 크기의 2차원 리스트 `self.graph`를 생성하고 모든 값을 0으로 초기화합니다.

### `add_edge(self, u, v)`

- **목적**: 두 정점 `u`와 `v` 사이에 간선을 추가합니다.
- **매개변수**:
  - `u`, `v`: 연결할 두 정점의 인덱스 (1부터 시작하는 인덱스라고 가정).
- **동작**:
  - 입력받은 정점 번호는 1부터 시작한다고 가정하므로, 내부적으로는 `u-1`, `v-1` 인덱스를 사용합니다.
  - 무향 그래프이므로 행렬의 대칭되는 두 위치 `[u-1][v-1]`과 `[v-1][u-1]`을 모두 `1`로 설정합니다.

### `show(self)`

- **목적**: 현재 그래프의 인접 행렬 상태를 출력합니다.
- **동작**:
  - 행렬의 각 행을 순회하며 요소를 출력합니다.

## 사용법

코드 하단에 포함된 예제를 통해 사용법을 확인할 수 있습니다.

```python
g = Graph(100)  # 100개의 정점을 가진 그래프 생성

# 간선 추가 (1-based index)
g.add_edge(1, 4)
g.add_edge(4, 2)
g.add_edge(4, 5)
g.add_edge(2, 5)
g.add_edge(5, 3)

# 인접 행렬 출력
g.show()
```

## 특징

- **공간 복잡도**: $O(V^2)$. 정점의 개수가 많아지면 메모리 사용량이 급격히 증가합니다.
- **시간 복잡도**:
  - 간선 추가: $O(1)$.
  - 두 정점의 연결 확인: $O(1)$.
  - 특정 정점의 모든 이웃 찾기: $O(V)$.
- **희소 그래프(Sparse Graph)**의 경우 인접 리스트 방식보다 메모리 낭비가 심할 수 있습니다.
