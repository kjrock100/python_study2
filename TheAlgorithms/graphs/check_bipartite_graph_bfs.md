# 이분 그래프 확인 (BFS) (Check Bipartite Graph BFS)

이 문서는 `check_bipartite_graph_bfs.py` 파일에 구현된 **BFS(너비 우선 탐색)**를 이용한 이분 그래프 판별 알고리즘에 대해 설명합니다.

## 개요

**이분 그래프(Bipartite Graph)**란 모든 정점을 두 개의 서로 겹치지 않는 집합(U, V)으로 나눌 수 있어, 그래프 내의 모든 간선이 U의 정점과 V의 정점을 연결하는 그래프를 말합니다. 다시 말해, 인접한 두 정점이 서로 다른 색으로 칠해질 수 있는 그래프(2-Colorable)입니다.

## 주요 함수: `checkBipartite`

### `checkBipartite(graph)`

- **목적**: 주어진 그래프가 이분 그래프인지 확인합니다.
- **매개변수**:
  - `graph`: 인접 리스트 형태의 딕셔너리 (예: `{0: [1, 3], 1: [0, 2], ...}`).
- **반환값**: 이분 그래프이면 `True`, 아니면 `False`.

### 알고리즘 동작 원리

1. **초기화**:
   - `visited`: 방문 여부를 저장하는 리스트.
   - `color`: 각 정점의 색상을 저장하는 리스트 (`-1`: 미방문, `0`: 색상 A, `1`: 색상 B).
   - `queue`: BFS 탐색을 위한 큐.

2. **전체 정점 순회**:
   - 그래프가 연결되어 있지 않을 수 있으므로, 모든 정점에 대해 반복문을 돌며 방문하지 않은 정점에서 BFS를 시작합니다.

3. **BFS 탐색 (`bfs` 내부 함수)**:
   - 시작 정점을 큐에 넣고 색상 `0`을 할당합니다.
   - 큐가 빌 때까지 다음을 반복합니다:
     - 정점 `u`를 꺼냅니다.
     - `u`의 모든 이웃 `neighbour`에 대해:
       - **Self-loop 검사**: `neighbour == u`이면 이분 그래프가 아닙니다.
       - **미방문 정점**: 아직 색이 칠해지지 않았다면(`-1`), 현재 정점 `u`와 반대되는 색(`1 - color[u]`)을 칠하고 큐에 넣습니다.
       - **색상 충돌 검사**: 이미 방문했는데 현재 정점 `u`와 같은 색이라면, 인접한 정점이 같은 색이므로 이분 그래프가 아닙니다 (`False` 반환).

4. **결과 반환**:
   - 모든 정점을 문제없이 탐색하고 색칠할 수 있다면 `True`를 반환합니다.

## 사용법

`if __name__ == "__main__":` 블록에서 실행 예시를 확인할 수 있습니다.

```python
graph_data = {
    0: [1, 3],
    1: [0, 2],
    2: [1, 3],
    3: [0, 2]
}
print(checkBipartite(graph_data))
# 출력: True
```
