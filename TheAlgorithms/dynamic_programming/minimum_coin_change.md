# 동전 거스름돈 개수 구하기 (Coin Change - Number of Ways)

이 문서는 `minimum_coin_change.py` 파일에 구현된 알고리즘에 대해 설명합니다. 파일명은 `minimum_coin_change`이지만, 실제 구현된 `dp_count` 함수는 주어진 동전들로 특정 금액을 만들 수 있는 **모든 방법의 수**를 계산하는 알고리즘입니다.

## 개요

무한히 사용할 수 있는 $m$가지 종류의 동전 $S = \{S_0, S_1, \dots, S_{m-1}\}$가 주어졌을 때, 이 동전들을 조합하여 목표 금액 $n$을 만드는 방법이 총 몇 가지인지 구하는 문제입니다. 이 문제는 **동적 계획법(Dynamic Programming)**을 사용하여 효율적으로 해결할 수 있습니다.

## 주요 함수: `dp_count(S, n)`

### `dp_count(S: list[int], n: int) -> int`
- **목적**: 동전 리스트 `S`를 사용하여 금액 `n`을 만드는 모든 방법의 수를 반환합니다.
- **매개변수**:
  - `S`: 사용할 수 있는 동전의 가치가 담긴 리스트.
  - `n`: 만들고자 하는 목표 금액.
- **반환값**: 목표 금액 `n`을 만드는 방법의 수. `n`이 음수이면 0을 반환합니다.

### 알고리즘 동작 원리 (Tabulation)

1. **테이블 초기화**:
   - `table` 리스트를 생성하여 `0`부터 `n`까지의 각 금액을 만드는 방법의 수를 저장합니다. 크기는 `n + 1`입니다.
   - `table[0] = 1`: 금액 0을 만드는 방법은 '아무 동전도 선택하지 않는' 1가지 경우가 존재합니다.

2. **반복 및 상태 전이**:
   - 각 동전(`coin_val`)에 대해 반복합니다.
   - 해당 동전의 가치부터 목표 금액 `n`까지(`j`) 순회하며 테이블을 갱신합니다.
   - **점화식**: `table[j] += table[j - coin_val]`
     - 금액 `j`를 만드는 방법의 수는, 기존에 구해진 방법의 수에 '현재 동전(`coin_val`)을 하나 추가하여 금액 `j`를 만드는 경우'를 더한 것입니다.
     - 즉, `j - coin_val` 금액을 만드는 방법의 수만큼 더해줍니다.

3. **결과 반환**:
   - 모든 동전에 대해 계산이 끝나면 `table[n]`을 반환합니다.

## 사용법

`if __name__ == "__main__":` 블록은 `doctest`를 실행하여 함수가 올바르게 동작하는지 검증합니다.

예시:
```python
dp_count([1, 2, 3], 4)
# 결과: 4
# 설명: {1,1,1,1}, {1,1,2}, {2,2}, {1,3} -> 총 4가지 방법
```
