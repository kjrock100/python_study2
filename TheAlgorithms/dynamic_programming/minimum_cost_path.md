# 최소 비용 경로 (Minimum Cost Path)

이 문서는 `minimum_cost_path.py` 파일에 구현된 **최소 비용 경로** 찾기 알고리즘에 대해 설명합니다. 이 문제는 주어진 행렬(Matrix)의 왼쪽 위(Top-Left)에서 오른쪽 아래(Bottom-Right)까지 이동하는 데 필요한 최소 비용을 찾는 고전적인 동적 계획법(Dynamic Programming) 문제입니다.

## 개요

이동은 오직 **오른쪽** 또는 **아래쪽**으로만 가능하다고 가정합니다. 알고리즘은 입력받은 행렬 자체를 DP 테이블로 사용하여, 각 셀에 도달하기까지의 누적 최소 비용을 계산하고 저장하는 방식으로 동작합니다.

## 주요 함수: `minimum_cost_path`

### `minimum_cost_path(matrix: list[list[int]]) -> int`
- **목적**: 주어진 행렬에서 왼쪽 위에서 오른쪽 아래까지 이동하는 최소 비용을 계산합니다.
- **매개변수**:
  - `matrix`: 각 셀의 비용이 담긴 2차원 리스트.
- **반환값**: 최종 목적지까지의 최소 비용 합계.

### 알고리즘 동작 원리

1. **첫 행 전처리 (Preprocessing the first row)**:
   - 첫 번째 행의 각 셀은 오직 왼쪽에서만 도달할 수 있습니다.
   - 따라서 첫 행을 왼쪽에서 오른쪽으로 순회하며 누적 합을 계산합니다.
   - `matrix[0][i] += matrix[0][i - 1]`

2. **첫 열 전처리 (Preprocessing the first column)**:
   - 첫 번째 열의 각 셀은 오직 위쪽에서만 도달할 수 있습니다.
   - 따라서 첫 열을 위에서 아래로 순회하며 누적 합을 계산합니다.
   - `matrix[i][0] += matrix[i - 1][0]`

3. **나머지 경로 비용 갱신 (Updating the rest of the path costs)**:
   - 나머지 셀(`matrix[i][j]`)에 도달하는 방법은 위쪽(`matrix[i-1][j]`)에서 오거나 왼쪽(`matrix[i][j-1]`)에서 오는 두 가지 경우가 있습니다.
   - **점화식**: `matrix[i][j] += min(matrix[i - 1][j], matrix[i][j - 1])`
   - 즉, 현재 셀의 비용에 '위쪽까지의 최소 비용'과 '왼쪽까지의 최소 비용' 중 더 작은 값을 더하여 갱신합니다.

4. **결과 반환**:
   - 모든 계산이 끝나면, 오른쪽 맨 아래 셀(`matrix[-1][-1]`)에는 시작점부터 해당 위치까지의 총 최소 비용이 저장되어 있으므로 이 값을 반환합니다.

## 사용법

`if __name__ == "__main__":` 블록은 `doctest`를 실행하여 함수가 올바르게 동작하는지 검증합니다.

```python
minimum_cost_path([[2, 1, 4], [2, 1, 3], [3, 2, 1]])
# 결과: 7
# 경로: 2 -> 1 -> 1 -> 2 -> 1, 합계: 7
```

## 참고 사항
- 이 구현은 입력으로 받은 `matrix`를 직접 수정합니다(in-place). 원본 데이터를 보존해야 할 경우, 함수 시작 시 행렬을 복사하여 사용해야 합니다.
