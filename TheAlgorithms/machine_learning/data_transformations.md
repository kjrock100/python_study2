# 데이터 변환 (Data Transformations)

이 문서는 `data_transformations.py` 파일에 구현된 **정규화(Normalization)**와 **표준화(Standardization)** 데이터 변환 기법에 대해 설명합니다.

## 개요

데이터 변환은 머신러닝 모델의 학습 속도를 높이고 성능을 향상시키기 위해 숫자형 데이터의 범위를 일정한 스케일로 맞추는 과정입니다. 이 파일은 가장 널리 사용되는 두 가지 방법인 정규화와 표준화를 구현합니다.

## 1. 정규화 (Normalization)

### `normalization(data: list, ndigits: int = 3) -> list`

- **목적**: 데이터의 모든 값을 0과 1 사이의 범위로 변환합니다.
- **공식**: $x_{norm} = \frac{x - x_{min}}{x_{max} - x_{min}}$
- **특징**:
  - 모든 데이터가 동일한 척도를 갖게 됩니다.
  - 일반적으로 데이터가 정규 분포(Gaussian Distribution)를 따르지 않을 때 더 좋은 성능을 보입니다.

#### 사용 예시
```python
normalization([2, 7, 10, 20, 30, 50])
# 결과: [0.0, 0.104, 0.167, 0.375, 0.583, 1.0]
```

## 2. 표준화 (Standardization)

### `standardization(data: list, ndigits: int = 3) -> list`

- **목적**: 데이터의 분포를 평균이 0이고 표준편차가 1인 표준 정규 분포로 변환합니다. **Z-점수 정규화(Z-score Normalization)**라고도 불립니다.
- **공식**: $x_{std} = \frac{x - \mu}{\sigma}$ (여기서 $\mu$는 평균, $\sigma$는 표준편차)
- **특징**:
  - 데이터에 이상치(Outlier)가 있을 때 정규화보다 더 안정적인 성능을 보입니다.
  - 데이터가 정규 분포를 따를 때 효과적입니다.

#### 사용 예시
```python
standardization([2, 7, 10, 20, 30, 50])
# 결과: [-0.999, -0.719, -0.551, 0.009, 0.57, 1.69]
```

## 정규화 vs 표준화 선택

두 방법 중 어느 것이 더 나은지는 데이터의 특성에 따라 달라지며, 실험을 통해 결정하는 것이 가장 좋습니다. 일반적인 가이드라인은 다음과 같습니다.

- **정규화**: 데이터가 정규 분포를 따르지 않을 때.
- **표준화**: 데이터가 정규 분포를 따르거나, 이상치가 포함되어 있을 때.

## 요구 사항
- `statistics`: 평균(`mean`)과 표준편차(`stdev`) 계산을 위해 사용됩니다.
